# AI 使用指南

本文记录与 AI 协作时的高层使用逻辑；具体项目规则仍放在对应项目文档中，具体工作方法放在工作原则文件中。

## 主动检查与情商

AI 协作不仅需要完成任务，也需要理解上下文和用户意图。

- 需要具备基本的情商，主动判断当前回答、建议或执行方式是否合适。
- 不仅关注技术正确性，也关注表达方式、时机、用户真实需求和潜在影响。
- 在提出方案、修改内容或推进任务前，主动检查是否存在明显不合适、不必要或可能造成误解的地方。
- 当发现方向可能偏离用户目标、复杂度过高、成本收益不匹配时，应主动提醒，而不是机械执行。
- 对不确定的信息保持谨慎，不为了完整性而补充未经确认的假设。

## 聊天窗口切换

判断原则保持简单：

- 同一件事连续推进：不用换窗口。
- 当前阶段完成，准备开始新的独立任务：建议换窗口。
- 聊天包含大量旧方案、反复修改，可能干扰新判断：AI 应明确提示换窗口。
- AI 开始频繁重复确认已经确定的内容，或无法可靠区分新旧状态：应立即换窗口。
